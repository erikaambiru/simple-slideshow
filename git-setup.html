Gitインストールとリポジトリクローンの手順

このガイドでは、Gitのインストールからリポジトリのクローンまでの手順を詳しく説明します。

1 Gitのインストール

Gitは、バージョン管理システムとして広く使われているツールです。以下の手順でインストールしましょう。

Windowsの場合

  1. Git for Windowsをダウンロードします。
  2. ダウンロードしたインストーラーを実行します。
  3. インストールオプションは基本的にデフォルトのままで問題ありませんが、以下の点に注意してください：
    • 「Adjusting your PATH environment」では「Git from the command line and also from 3rd-party software」を選択すると便利です。
    • 「Configuring the line ending conversions」では、Windowsの場合「Checkout Windows-style, commit Unix-style line endings」がおすすめです。
  4. インストールが完了したら、コマンドプロンプトまたはPowerShellを開き、git --versionと入力して、Gitが正しくインストールされたか確認します。

macOSの場合

  1. Homebrewを使う方法（推奨）：
    brew install git
  2. インストーラーを使う方法： Git for Macからインストーラーをダウンロードして実行します。
  3. ターミナルを開き、git --versionと入力して、Gitが正しくインストールされたか確認します。

Linuxの場合

ディストリビューションによって異なりますが、一般的なコマンドは以下の通りです：

  • Debian/Ubuntu：
    sudo apt-get update
    sudo apt-get install git
  • Fedora：
    sudo dnf install git
  • CentOS/RHEL：
    sudo yum install git
  • インストール後、git --versionでインストールを確認します。
ヒント

Gitをインストールすると、Git Bash（Windowsの場合）やターミナルからGitコマンドを実行できるようになります。また、多くのIDEやテキストエディタにはGit統合機能があり、GUIでGitを操作することも可能です。

2 Gitの初期設定

Gitをインストールしたら、ユーザー名とメールアドレスを設定します。これは、あなたが行った変更を識別するために使用されます。

git config --global user.name "あなたの名前"
git config --global user.email "あなたのメールアドレス"

設定を確認するには、以下のコマンドを実行します：

git config --list
注意

GitHubなどのサービスを使用する場合は、登録したメールアドレスと同じものを設定することをおすすめします。

3 GitHubアカウントの作成（オプション）

リモートリポジトリを使用する場合は、GitHubなどのサービスのアカウントが必要です。GitHubアカウントの作成手順は以下の通りです：

  1. GitHubにアクセスします。
  2. 「Sign up」ボタンをクリックします。
  3. メールアドレス、パスワード、ユーザー名を入力します。
  4. アカウント確認のためのメールを受け取り、確認します。
  5. アカウント設定を完了します。
ヒント

GitHubでは、無料アカウントでも無制限の公開リポジトリを作成できます。また、プライベートリポジトリも制限付きで利用可能です。

4 SSH鍵の設定（推奨）

GitHubなどのサービスとの安全な通信のために、SSH鍵を設定することをおすすめします。

SSH鍵の生成

  1. ターミナル（Git Bash、コマンドプロンプト、PowerShellなど）を開きます。
  2. 以下のコマンドを実行して、SSH鍵を生成します：
    ssh-keygen -t ed25519 -C "あなたのメールアドレス"
  3. 保存先の指定を求められたら、デフォルトのままEnterキーを押します。
  4. パスフレーズの入力を求められたら、セキュリティを高めるためにパスフレーズを設定することをおすすめします。

SSH鍵をGitHubに追加

  1. 以下のコマンドで公開鍵をクリップボードにコピーします：
    # Windowsの場合
    clip < ~/.ssh/id_ed25519.pub

    # macOSの場合
    pbcopy < ~/.ssh/id_ed25519.pub

    # Linuxの場合
    xclip -sel clip < ~/.ssh/id_ed25519.pub
  2. GitHubにログインし、右上のプロフィールアイコンをクリックして「Settings」を選択します。
  3. 左側のメニューから「SSH and GPG keys」を選択します。
  4. 「New SSH key」ボタンをクリックします。
  5. タイトルを入力し（例：「My Laptop」）、「Key」フィールドに先ほどコピーした公開鍵を貼り付けます。
  6. 「Add SSH key」ボタンをクリックして保存します。
注意

SSH鍵を設定すると、パスワードを入力せずにGitHubとの通信が可能になります。ただし、パスフレーズを設定した場合は、最初の接続時にパスフレーズの入力が必要です。

5 リポジトリのクローン

既存のリポジトリをローカルにコピー（クローン）する方法を説明します。

HTTPSを使用したクローン

  1. GitHubのリポジトリページにアクセスします。
  2. 「Code」ボタンをクリックし、「HTTPS」タブを選択します。
  3. URLをコピーします（例：https://github.com/username/repository.git）。
  4. ターミナルを開き、リポジトリをクローンしたいディレクトリに移動します。
  5. 以下のコマンドを実行します：
    git clone https://github.com/username/repository.git
  6. GitHubのユーザー名とパスワード（またはアクセストークン）の入力を求められる場合があります。

SSHを使用したクローン（SSH鍵を設定済みの場合）

  1. GitHubのリポジトリページにアクセスします。
  2. 「Code」ボタンをクリックし、「SSH」タブを選択します。
  3. URLをコピーします（例：git@github.com:username/repository.git）。
  4. ターミナルを開き、リポジトリをクローンしたいディレクトリに移動します。
  5. 以下のコマンドを実行します：
    git clone git@github.com:username/repository.git

スライドショーリポジトリのクローン

このスライドショーのリポジトリをクローンするには、以下のコマンドを実行します：

# HTTPSを使用する場合
git clone https://github.com/erikaambiru/simple-slideshow.git

# SSHを使用する場合
git clone git@github.com:erikaambiru/simple-slideshow.git
ヒント

クローンしたリポジトリには、すべてのファイルとGit履歴が含まれています。リポジトリのディレクトリに移動して、git logコマンドを実行すると、コミット履歴を確認できます。

6 基本的なGitコマンド

リポジトリをクローンした後、以下の基本的なGitコマンドを使って作業を進めることができます：

  • git status：現在の作業ディレクトリの状態を確認します。
    git status
  • git add：変更をステージングエリアに追加します。
    # 特定のファイルを追加
    git add filename.txt

    # すべての変更を追加
    git add .
  • git commit：ステージングエリアの変更をコミットします。
    git commit -m "コミットメッセージ"
  • git pull：リモートリポジトリの変更をローカルリポジトリに取り込みます。
    git pull
  • git push：ローカルの変更をリモートリポジトリに送信します。
    git push
  • git branch：ブランチを一覧表示します。
    git branch
  • git checkout：ブランチを切り替えます。
    # 既存のブランチに切り替え
    git checkout branch-name

    # 新しいブランチを作成して切り替え
    git checkout -b new-branch-name
注意

最近のGitでは、git checkoutの代わりにgit switchとgit restoreを使うことが推奨されています。

7 トラブルシューティング

Gitを使用する際によく遭遇する問題とその解決方法を紹介します。

認証エラー

GitHubへのプッシュ時に認証エラーが発生する場合：

  • HTTPSを使用している場合は、正しいユーザー名とパスワード（またはアクセストークン）を入力しているか確認してください。
  • SSHを使用している場合は、SSH鍵が正しく設定されているか確認してください。

マージ競合

複数の人が同じファイルを編集すると、マージ競合が発生することがあります。競合を解決するには：

  1. 競合しているファイルを開きます。
  2. 競合マーカー（<<<<<<<、=======、>>>>>>>）を探します。
  3. 競合を手動で解決し、マーカーを削除します。
  4. 変更をステージングし、コミットします。

コミットの取り消し

直前のコミットを取り消す方法：

# コミットを取り消すが、変更は保持する
git reset --soft HEAD~1

# コミットを取り消し、変更も破棄する
git reset --hard HEAD~1
ヒント

Gitに関する詳細なドキュメントは、公式ドキュメントで確認できます。また、Oh Shit, Git!?は、Gitで困ったときに役立つリソースです。

8 おすすめのGitツール

コマンドラインでのGit操作が苦手な場合は、以下のGUIツールを検討してみてください：

  • GitHub Desktop： GitHubが提供する無料のデスクトップクライアント。シンプルで使いやすいインターフェースが特徴です。
  • Sourcetree： Atlassianが提供する無料のGitクライアント。高度な機能を備えています。
  • GitKraken： 美しいインターフェースと直感的な操作が特徴のGitクライアント。基本機能は無料で利用できます。
  • Visual Studio Code： 人気のコードエディタで、Gitの統合機能が充実しています。
ヒント

GUIツールを使用すると、コマンドラインを使わずにGitの操作が可能ですが、基本的なGitコマンドを理解しておくことをおすすめします。
