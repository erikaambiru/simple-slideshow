AI用語集について

このページでは、AI勉強会で使用される主要な用語を解説しています。AI開発や利用の際に役立つ基本的な用語から応用的な概念まで幅広く網羅しています。

基本用語
用語 説明
AI（人工知能） 人間の知能を模倣するよう設計されたシステム。機械学習やディープラーニングも含まれる広い概念。
機械学習（Machine Learning） 経験（データ）をもとにパターンを学び、予測や分類を行うAIの一分野。
ディープラーニング（Deep Learning） 人間の脳を模した「ニューラルネットワーク」を使って高度な学習を行う機械学習の手法。画像認識や自然言語処理で活躍。
プロンプト（Prompt） AIに対する指示・入力文。特に生成AI（ChatGPTなど）で使われる言葉。
自然言語処理（NLP） 人間の言語（自然言語）を理解・生成・翻訳・要約する技術。ChatGPTもこの技術を使っている。
生成AI関連
用語 説明
トークン（Token） AIが処理する最小単位のテキスト。日本語なら1〜2文字、英語なら単語程度。処理制限や料金計算の単位。
温度（Temperature） 出力のランダム性を調整するパラメータ。高いと創造的、低いと堅実な回答になりやすい。
ファインチューニング 既存のAIモデルを特定の目的・データに合わせて追加学習させること。
AIエージェント 自律的に目標達成のために行動を選ぶAIの仕組み。複数のツールやAPIと連携するものも多い。
AI関連技術
用語 説明
API 外部アプリケーションからAIの機能を利用するためのインターフェース。OpenAI APIを使えば、自分のアプリにChatGPTを組み込める。
Embeddings（埋め込み） 単語や文章を数値ベクトルに変換し、意味的な類似度などを計算できるようにする技術。検索や分類に活用。
RAG（Retrieval-Augmented Generation） 検索（情報取得）＋生成を組み合わせたAIの手法。企業データやFAQなどに基づいて自然な回答を出せる。
マルチモーダルAI テキストだけでなく、画像・音声・動画など複数のデータ形式を扱うAI（例：GPT-4の画像入力機能など）。
応用・実践用語
用語 説明
プロンプトエンジニアリング AIに適切な応答をさせるために、入力文を工夫・調整する技術。
ハルシネーション AIがそれらしく見えるが実際には間違った内容を生成する現象。「それっぽいウソ」に注意。
コーパス（Corpus） AIの学習に使われる大量の文章・データ集。
LLM（大規模言語モデル） 巨大なテキストデータから学習し、多用途に使える自然言語モデル（例：GPT-4、Claude、Geminiなど）。
ファインチューニング vs プロンプト設計 モデル自体をカスタマイズする方法と、入力（プロンプト）を工夫する方法の対比。前者は高コスト、後者は手軽。
主要な生成AIモデルと提供企業
モデル名 提供企業 概要
Claude Anthropic（アンソロピック） 「安全で信頼性のあるAI」を重視する企業が開発した対話型AI。倫理性と制御可能性に重点を置いており、Claude 3は多言語対応や文脈理解に優れる。
GPT（ChatGPT） OpenAI GPT（Generative Pre-trained Transformer）シリーズの最新版はGPT-4（ProはGPT-4-turbo）。自然な対話や生成性能に優れており、世界中で最も広く使われている対話AIの一つ。
Gemini（旧Bard） Google DeepMind（グーグル・ディープマインド） Googleが開発した大規模言語モデル。Bardから名前を変更し、マルチモーダル（テキスト＋画像など）対応が特徴。GmailやGoogleドキュメントと連携可能。
Grok xAI（イーエックスエーアイ）（※創設者：イーロン・マスク） X（旧Twitter）に統合されている対話AI。皮肉やユーモアを理解しやすいように設計されており、やや"人間臭い"応答が特徴。
DeepSeek DeepSeek（中国発のAIスタートアップ） オープンソース型の大規模言語モデルで、英語と中国語に強い。研究・技術者向けの活用が進んでいる。高精度かつパフォーマンスに優れた構造が売り。
